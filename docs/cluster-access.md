# Cluster Access

The homelab Kubernetes cluster runs on **Talos Linux** with three
control-plane nodes (`talos-cp{1,2,3}`) reachable on the VRRP virtual IP
`192.168.10.245:6443`.

The kubeconfig and talosconfig live in the sibling
[`homelab-infrastructure`](../../homelab-infrastructure) repository under
`talos/`. They are generated by OpenTofu when the cluster is bootstrapped.

## 1. Quick-start (recommended)

The Nix flake in the sibling repo provides a `talos` developer shell that:

- pre-installs `talosctl`, `kubectl`, `k9s`, `helm`, `fluxcd`, `sops`, `age`
- exports `KUBECONFIG` and `TALOSCONFIG` automatically
- aliases `k=kubectl`

```bash
cd ../homelab-infrastructure
nix develop .#talos

# Verify
kubectl get nodes
flux get kustomizations -A
talosctl --nodes 192.168.10.41 health
```

## 2. Manual access (without Nix)

```bash
export KUBECONFIG=$(realpath ../homelab-infrastructure/talos/kubeconfig)
export TALOSCONFIG=$(realpath ../homelab-infrastructure/talos/talosconfig)
kubectl get nodes
```

The kubeconfig embeds the cluster CA + admin client certificate; treat
it as a privileged credential. **Do not commit your local copy** —
it is generated by Terraform and the canonical version is under that
state.

## 4. SOPS / age (decrypting secrets locally)

Encrypted secrets in this repo (anything matching `*.secret.yaml`) are
encrypted with the cluster's age public key (see `.sops.yaml`).

To decrypt locally you need the **private** age key. It lives in the
cluster as `Secret/flux-system/sops-age` and the offline copy is on
the operator workstation only.

```bash
# Either rely on the `age` shell:
nix develop .#age

# … or set the key file directly
export SOPS_AGE_KEY_FILE=~/.config/sops/age/keys.txt
sops -d infrastructure/base/sources/hetzner.secret.yaml
```

For day-to-day secret operations there are `just` helpers in this repo
(`just sops-edit`, `just sops-create`, `just barman-s3-credentials`, …).

## 5. Talos administration

```bash
nix develop ../homelab-infrastructure#talos

# Cluster-wide
talosctl --nodes 192.168.10.41,192.168.10.42,192.168.10.43 health

# Read live machine config of one node
talosctl --nodes 192.168.10.41 get machineconfig

# Apply a config change (use Terraform for permanent changes)
talosctl apply-config --nodes 192.168.10.41 --file new-config.yaml
```

For everything cluster-config-related, prefer `tofu apply` in
`homelab-infrastructure/talos/` — Terraform is the source of truth for
the machine configuration; `talosctl apply-config` is for debug.

## 6. Day-to-day flux commands

```bash
# State of all Flux objects
flux get all -A

# Reconcile after a Git push (no need to wait for the next 1h interval)
flux reconcile source git flux-system
flux reconcile kustomization apps --with-source

# Tail Flux controller logs
kubectl -n flux-system logs -lapp=source-controller       -f
kubectl -n flux-system logs -lapp=kustomize-controller    -f
kubectl -n flux-system logs -lapp=helm-controller         -f
```

## 7. Troubleshooting

| Symptom                                              | Where to look                                |
|------------------------------------------------------|----------------------------------------------|
| Flux `Kustomization` stuck in `Reconciling`          | `flux events`, then `kubectl describe …`     |
| HelmRelease keeps reinstalling                       | `flux events --for HelmRelease/<name>`       |
| Pod `ImagePullBackOff`                               | `kubectl describe pod` — registry / TLS      |
| PVC `Forbidden: spec is immutable`                   | Stale Longhorn PVC, see `docs/applications.md` |
| Cert-manager `Order` failing                         | `kubectl describe order -A` + Hetzner DNS    |
